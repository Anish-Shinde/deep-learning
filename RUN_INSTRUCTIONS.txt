CSV Data Analyzer ‚Äî Simple Run Instructions
=============================================

What is this?
-------------
A web-based tool to analyze, clean, and build machine learning models with your CSV data files. No coding required - just upload your data and click buttons!

Quick Start (5 Simple Steps)
-----------------------------

Step 1: Check Python Installation
Open PowerShell and type:
```powershell
python --version
```
‚úÖ You should see something like "Python 3.8.x" or higher. If not, install Python from python.org first.

Step 2: Open Project Folder
Navigate to the project folder in PowerShell:
```powershell
cd "C:\Users\Aarav Comp\Desktop\submission\deep learning"
```

Step 3: Create Virtual Environment
This keeps the project dependencies separate:
```powershell
python -m venv .venv
```

‚ö†Ô∏è **Important**: If you see an error about a path mismatch (like "CSV Aalyzer" when you're in "deep learning"), delete the old `.venv` folder first:
```powershell
Remove-Item -Recurse -Force .venv
python -m venv .venv
```

Step 4: Activate Virtual Environment
```powershell
.\.venv\Scripts\Activate.ps1
```
üí° If you see an error about execution policy, run this first (then try again):
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
.\.venv\Scripts\Activate.ps1
```

‚úÖ **Check**: After activation, you should see `(.venv)` at the start of your command line. This means it's activated correctly!

Step 5: Install Dependencies & Run
```powershell
pip install -r requirements.txt
streamlit run csv_analyzer.py
```

**Alternative Method (More Reliable):**
If the above doesn't work, you can run Streamlit directly using the virtual environment's Python:
```powershell
pip install -r requirements.txt
.\.venv\Scripts\python.exe -m streamlit run csv_analyzer.py
```
This method doesn't require activating the virtual environment first.

üéâ That's it! Your browser should open automatically. If not, look for a URL like http://localhost:8501

‚ö†Ô∏è **Important Notes**:
- Make sure you're in the correct folder (the one containing `csv_analyzer.py`)
- The virtual environment (`.venv`) must be in the same folder as your project
- Always activate the virtual environment before running the app (you'll see `(.venv)` in your command line)

Common Issues & Solutions
-------------------------

‚ùå **"python is not recognized"**
   ‚Üí Install Python from python.org and make sure to check "Add Python to PATH" during installation

‚ùå **"streamlit is not recognized"**
   ‚Üí Try using the direct Python method instead:
   ```powershell
   .\.venv\Scripts\python.exe -m streamlit run csv_analyzer.py
   ```
   Or make sure you activated the virtual environment (Step 4). You should see `(.venv)` at the start of your command line.

‚ùå **"Activation script cannot be loaded"**
   ‚Üí Run this command first:
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```
   Then try activating again.

‚ùå **"Fatal error in launcher" or "Unable to create process"**
   ‚Üí This means your virtual environment was created in a different folder. Fix it:
   ```powershell
   Remove-Item -Recurse -Force .venv
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1
   pip install -r requirements.txt
   ```
   Then run using the direct Python method:
   ```powershell
   .\.venv\Scripts\python.exe -m streamlit run csv_analyzer.py
   ```

‚ùå **"App runs but shows errors"**
   ‚Üí Make sure you installed all dependencies with: `pip install -r requirements.txt`

‚ùå **App is slow**
   ‚Üí Large datasets or complex models take time. Be patient, or try with a smaller sample of your data first.

üí° **Tips**: 
- Keep the PowerShell window open while using the app. Closing it will stop the application.
- If you're having trouble with activation, use the direct Python method: `.\.venv\Scripts\python.exe -m streamlit run csv_analyzer.py`
- To check if the app is running, look for Python processes or open http://localhost:8501 in your browser

Deploying to Streamlit Cloud
-----------------------------
If you're using Streamlit Cloud (streamlit.io) connected to your GitHub repository:

**When Updates Appear:**
- After pushing code to GitHub, Streamlit Cloud automatically detects changes
- Redeployment usually takes **1-3 minutes** after your push completes
- You'll see a "Redeploying..." status in your Streamlit Cloud dashboard
- The app will automatically update once deployment finishes

**How to Check Deployment Status:**
1. Go to your Streamlit Cloud dashboard (share.streamlit.io)
2. Look for your app - it will show "Redeploying..." or "Deployed"
3. Click on your app to see deployment logs and status
4. Green checkmark = successfully deployed
5. Red X = deployment failed (check logs for errors)

**Manual Redeploy:**
- In Streamlit Cloud dashboard, click "‚ãÆ" (three dots) next to your app
- Select "Redeploy" to manually trigger a new deployment

**Troubleshooting Deployment:**
- If changes don't appear after 5 minutes, check:
  - Did you push to the correct branch? (usually `main` or `master`)
  - Are there any errors in the deployment logs?
  - Is your `requirements.txt` up to date with all dependencies?
  - Check that `csv_analyzer.py` is in the root directory

**Note:** The first deployment may take longer (5-10 minutes) as it installs all dependencies.
